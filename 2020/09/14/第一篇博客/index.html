我的博客创建之路

#创建初衷
一开始有创建博客的想法是在B站看到了codesheep的创建博客的视频就想创建一个自己的博客，记录自己每天的学习历程
从而起到监督自己的作用，但是创建的过程出现了许多的曲折下面来记录一下我的博客创建之路

##我使用的是hexo方式创建博客
###前提要想使用hexo创建自己的博客首先要安装node.js，git,github上创建自己的账号
在三个条件都完成的情况下我们开始我们的创建之路

##创建之路
在三个软件都安装成功之后我们打开cmd查看是否安装成功

node -v #查看node版本

npm -v #查看npm版本
git version查看git是否安装成功
安装淘宝镜像npm install -g cnpm –registry=http://registry.npm.taobao.org
安装完成后查看我们的cnpm版本 cnpm -v
如果显示cnpm安装成功安装hexo框架cnpm install -g hexo-cli
设置博客安装的目录 hexo init floder
cd floder进入创建的博客目录下
npm install
注：floder是你想创建的hexo目录 自己决定

##在GitHub创建代码仓库
###点击右上角 + new repository 创建仓库
只填写bolgname.github.io即可，然后点击create repositrory。
注： blogname.github.io 的blogname要和用户名保持一致，不然后面会失败
点击github主页点击头像下面的profile,找到新建立的username.github.io文件打开，点击settings，往下拉动鼠标到GitHub Page
点击choose a theme，随便选择一个，然后select theme保存就行了

#配置ssh密匙
首先查看是否存在ssh密匙
cd ~/. ssh
如果出现bash: -keygen: command not found
输入export PATH=/usr/bin:$PATH
接着继续尝试cd ~/. ssh
如果出现**.ssh: No such file or directory**
则输入ssh-keygen -t rsa -C +Email
出现一大段文字时尝试ssh-add ~/.ssh/id_rsa
如果出现command not found
则输入ssh-agent bash之后继续尝试ssh-add ~/.ssh/id_rsa
注：以上错误如果没出现所提的错误请忽略
cat ~/.ssh/id_rsa.pub查询ssh密匙赋值出现的一大段文字打开GitHub点击右上角头像然后点击setting
点击SSH AND GPG Keys 然后选择sshkeys加入链接

##测试
完成以上操作后在gitbash中输入ssh -T git@github.com进行测试
如果成功会出现:The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?
第一次会出现时候countinue 输入yes点击回车

设置用户信息
git config –global user.name “你的用户名”//用户名
git config –global user.email “你的邮箱”//填写自己的邮箱

提示成功后,打开Hexo文件夹下的_config.yml，这个是博客的配置文件，添加你的GitHub page url应该是在第16行的url地址中填写你的GitHub page ur 就行
然后执行hexo g && hexo d 或者 hexo g -d
此时在浏览器打开你的主页就可以看到你的博客了 主页就是你的github page url
